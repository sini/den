<Aside type="caution" title="Deprecation Notice">
Several `den.lib` functions are deprecated. The pipeline now handles parametric dispatch
and class module wrapping automatically. Functions marked **Deprecated** below still work
but should not be used in new code. See [Class Modules](/explanation/class-modules/).
</Aside>

## `den.lib.parametric`

**Deprecated.** Wraps an aspect with a `__functor` that filters `includes` by argument compatibility.
The pipeline now handles this automatically. See [Class Modules](/explanation/class-modules/) and [Parametric Aspects](/explanation/parametric/).

```nix
den.lib.parametric { nixos.x = 1; includes = [ ... ]; }
```

Default uses `atLeast` matching.

### `den.lib.parametric.atLeast`

Same as `parametric`. Functions match if all required params are present.

### `den.lib.parametric.exactly`

Functions match only if required params exactly equal provided params.

```nix
den.lib.parametric.exactly { includes = [ ({ host }: ...) ]; }
```

### `den.lib.parametric.fixedTo`

Calls the aspect with a fixed context, ignoring the actual context:

```nix
den.lib.parametric.fixedTo { host = myHost; } someAspect
```

### `den.lib.parametric.expands`

Extends the received context with additional attributes before dispatch:

```nix
den.lib.parametric.expands { extra = true; } someAspect
```

## `den.lib.canTake`

**Deprecated.** Function argument introspection.
The pipeline now handles this automatically. See [Class Modules](/explanation/class-modules/) and [Parametric Aspects](/explanation/parametric/).

### `den.lib.canTake params fn`

Returns `true` if `fn`'s required arguments are satisfied by `params` (atLeast).

### `den.lib.canTake.atLeast params fn`

Same as `canTake`.

### `den.lib.canTake.exactly params fn`

Returns `true` only if `fn`'s required arguments exactly match `params`.

## `den.lib.take`

**Deprecated.** Conditional function application.
The pipeline now handles this automatically. See [Class Modules](/explanation/class-modules/) and [Parametric Aspects](/explanation/parametric/).

### `den.lib.take.atLeast fn ctx`

Calls `fn ctx` if `canTake.atLeast ctx fn`, otherwise returns `{}`.

### `den.lib.take.exactly fn ctx`

Calls `fn ctx` if `canTake.exactly ctx fn`, otherwise returns `{}`.

### `den.lib.take.unused`

`_unused: used: used` -- ignores first argument, returns second. Used for
discarding `aspect-chain` in `import-tree`.

## Context Shortcuts

**Deprecated.** These helpers are shortcuts built with `den.lib.take.exactly` and
`den.lib.parametric.fixedTo`. The pipeline now handles this automatically. See [Class Modules](/explanation/class-modules/) and [Parametric Aspects](/explanation/parametric/).

### `den.lib.perHost aspect`

Run `aspect` only in `{ host }` contexts.

### `den.lib.perUser aspect`

Run `aspect` only in `{ host, user }` contexts.

### `den.lib.perHome aspect`

Run `aspect` only in `{ home }` contexts.

## `den.lib.__findFile`

The angle bracket resolver. See [Angle Brackets Syntax](/guides/angle-brackets/).

```nix
_module.args.__findFile = den.lib.__findFile;
```

## `den.lib.aspects`

Den aspects API. Provides aspect type definitions and internal resolution helpers.

### `den.lib.aspects.resolve class aspect`

**Internal.** Resolves an aspect for a given class (e.g., `"nixos"`), returning a
module with `imports`. This function is used by the fx pipeline and is not a
public API. Prefer using `entity.resolved` or the fx pipeline directly.

### Aspect filtering

Aspect filtering (formerly `den.lib.aspects.adapters`) is now handled
internally by the fx pipeline. The `adapters` API has been removed.

## `den.lib.resolveStage`

### `den.lib.resolveStage name ctx`

Resolves a stage by name with a given context attrset. Returns an aspect-like
record containing the stage's `provides`, `includes`, class attrs, and pipeline
metadata (`__ctx`, `__ctxStage`, `__scopeHandlers`). Used internally by the
pipeline and by `den.lib.diag` convenience wrappers.

```nix
root = den.lib.resolveStage "user" { inherit host user; };
```

## `den.lib.policyInspect`

### `den.lib.policyInspect.inspect { kind, context }`

Lightweight policy inspection utility. Calls resolve functions directly
without running the full pipeline. Returns matching policies with their
targets and routing type.

See [den.policies reference](/reference/policies/) for full documentation.
