## What is a Parametric Aspect

A **parametric aspect** is an aspect defined as a function whose arguments are
pipeline context values — `host`, `user`, `home`, or any custom entity kind.
Den introspects the function's argument pattern and only calls it in contexts
where all required arguments are available.

```nix
# This aspect only activates in {host} contexts
den.aspects.networking = { host, ... }: {
  nixos.networking.hostName = host.name;
};

# This only activates when both {host, user} are present
den.aspects.user-groups = { host, user, ... }: {
  nixos.users.users.${user.userName}.extraGroups = [ "wheel" ];
};

# This only activates for standalone {home} contexts
den.aspects.shell-config = { home, ... }: {
  homeManager.programs.zsh.enable = true;
};
```

No wrapper needed — a bare function requiring `{ host, user }` is silently
skipped in contexts that only have `{ host }`. The argument shape **is** the
condition. No `mkIf`, no `enable` flags.

Static attrsets (non-functions) are always included regardless of context:

```nix
den.aspects.firewall = {
  nixos.networking.firewall.enable = true;
};
```

## Where Parametric Args Work

<Aside type="tip" title="Aspect level only">
Parametric context args (`host`, `user`, `home`) work at the **aspect level** —
the outer function that wraps the entire aspect definition. They are pipeline
values, not NixOS module arguments.

For accessing Den context **inside** class modules (alongside `config`, `pkgs`, etc.),
use the [flat-form class module](/explanation/class-modules/) pattern instead.
</Aside>

**Aspect-level parametric** — the function wraps the whole aspect:

```nix
# host is a pipeline arg — the whole aspect is parametric
den.aspects.laptop = { host, ... }: {
  nixos.networking.hostName = host.name;
  homeManager.programs.git.userName = host.hostName;
};
```

**Class-level context injection** — Den args inside a class module:

```nix
# host here is pre-applied by wrapClassModule, not parametric dispatch
den.aspects.laptop = {
  nixos = { host, config, pkgs, ... }: {
    networking.hostName = host.name;
  };
};
```

Both patterns are valid and composable. Use aspect-level parametric when the
context drives the *entire aspect* (which classes to include, what includes to
add). Use class-level injection when only a specific class module needs
entity data alongside module-system args.

## The `canTake` Mechanism

`den.lib.canTake` is the underlying function that powers automatic dispatch.
It checks whether a function's required arguments are satisfiable:

```nix
canTake { host = ...; } ({ host, ... }: ...)  # => true
canTake { host = ...; } ({ host, user }: ...)  # => false (user missing)

canTake.exactly { host = ...; } ({ host }: ...)                 # => true
canTake.exactly { host = ...; foo = ...; } ({ host, ... }: ...) # => false
```

- `canTake.atLeast` — all required params present (default behavior).
- `canTake.exactly` — required params match provided params exactly.

You rarely need `canTake` directly — Den applies it automatically during
pipeline resolution.

<Aside type="caution" title="Avoid anonymous functions in includes">
Use named aspects instead of inline anonymous functions. Named aspects produce
better error traces and are easier to debug with the [diagram library](/explanation/diagrams/).

```nix
# Prefer this
den.aspects.my-laptop.includes = [ den.aspects.ssh-config ];
den.aspects.ssh-config = { host, ... }: { ... };

# Avoid this
den.aspects.my-laptop.includes = [
  ({ host, ... }: { ... })
];
```
</Aside>

## Deprecated Wrapper APIs

The following wrappers are no longer needed. Bare functions work directly —
Den handles dispatch automatically.

<Aside type="caution" title="Deprecated">
`den.lib.parametric`, `den.lib.perHost`, `den.lib.perUser`, `den.lib.perHome`,
and `den.lib.take.*` still exist for backward compatibility but should not
be used in new code.

```nix
# Old — den.lib.parametric wrapper
den.aspects.battery = den.lib.parametric {
  nixos.foo = 1;
  includes = [ ({ host, ... }: { nixos.x = host.name; }) ];
};

# New — bare parametric function
den.aspects.battery = { host, ... }: {
  nixos.foo = 1;
  nixos.x = host.name;
};
```

```nix
# Old — den.lib.perHost / perUser / perHome
den.lib.perHost ({ host, ... }: { nixos.x = host.name; })

# New — bare function (identical behavior)
{ host, ... }: { nixos.x = host.name; }
```
</Aside>
