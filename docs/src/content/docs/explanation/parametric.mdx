## Automatic Shape Detection

Den inspects function arguments at evaluation time using `builtins.functionArgs`.
A function's **required parameters** determine which contexts it matches --
no wrapper needed:

```nix
# Runs only when a {host} context exists
den.aspects.networking = { host, ... }: {
  nixos.networking.hostName = host.name;
};

# Runs only when both {host, user} are present
den.aspects.user-groups = { host, user, ... }: {
  nixos.users.users.${user.userName}.extraGroups = [ "wheel" ];
};

# Runs only for standalone {home} contexts
den.aspects.shell-config = { home, ... }: {
  homeManager.programs.zsh.enable = true;
};
```

A bare function requiring `{ host, user }` is silently skipped in contexts
that only have `{ host }`. No conditionals, no `mkIf`, no `enable` options --
the argument shape **is** the condition.

Static attrsets (non-functions) are always included regardless of context:

```nix
den.aspects.firewall = {
  nixos.networking.firewall.enable = true;
};
```

<Aside title="Class modules also support context injection">
Class modules can receive Den context directly in flat form:
`{ nixos = { host, config, ... }: ...; }` -- Den args are pre-applied
automatically before the module system sees them.
</Aside>

## The `canTake` Function

`den.lib.canTake` (`nix/fn-can-take.nix`) is the underlying mechanism that
powers automatic dispatch. It checks whether a function's required arguments
are satisfiable with the given context:

```nix
canTake { host = ...; } ({ host, ... }: ...)  # => true  (atLeast)
canTake { host = ...; } ({ host, user }: ...)  # => false (user missing)

canTake.exactly { host = ...; } ({ host }: ...)                    # => true
canTake.exactly { host = ...; foo = ...; } ({ host, ... }: ...)    # => false
```

- `canTake.atLeast` -- all required params are present in the context (default).
- `canTake.exactly` -- required params match provided params exactly, no extras.

You rarely need to call `canTake` directly. Den applies it automatically when
resolving aspects through the pipeline.

<Aside type="caution" title="Anonymous functions are an anti-pattern">
Avoid inlining anonymous functions in `includes` lists. Use named aspects
instead -- this improves readability and produces better error traces from Nix,
since the functions have a proper name and source location.

```nix
den.aspects.my-laptop.includes = [ den.aspects.foo ];
den.aspects.foo = { host, ... }: { ... };
```
</Aside>

## Deprecated Wrapper APIs

The following wrappers are no longer needed. Bare functions work directly as
aspects and includes -- Den handles dispatch automatically.

<Aside type="caution" title="Deprecated: den.lib.parametric">
Replace `den.lib.parametric { ... }` with a plain aspect. Parametric dispatch
is now built in.

```nix
# Old
den.aspects.battery = den.lib.parametric {
  nixos.foo = 1;
  includes = [
    ({ host, ... }: { nixos.x = host.name; })
  ];
};

# New -- just a context-aware function
den.aspects.battery = { host, ... }: {
  nixos.foo = 1;
  nixos.x = host.name;
};
```
</Aside>

<Aside type="caution" title="Deprecated: den.lib.perHost / den.lib.perUser / den.lib.perHome">
These wrappers added no behavior beyond what bare functions already provide.

```nix
# Old
den.lib.perHost ({ host, ... }: { nixos.x = host.name; })
den.lib.perUser ({ host, user, ... }: { nixos.x = user.userName; })
den.lib.perHome ({ home, ... }: { homeManager.x = true; })

# New -- bare functions, identical behavior
{ host, ... }: { nixos.x = host.name; }
{ host, user, ... }: { nixos.x = user.userName; }
{ home, ... }: { homeManager.x = true; }
```
</Aside>

<Aside type="caution" title="Deprecated: den.lib.take.atLeast / den.lib.take.exactly">
If you need explicit dispatch control, use `canTake` directly in rare cases.
For normal aspects, automatic detection is sufficient.

```nix
# Old
den.lib.take.atLeast fn ctx
den.lib.take.exactly fn ctx

# New -- typically unnecessary; den dispatches automatically
# If explicit control is needed:
if den.lib.canTake ctx fn then fn ctx else {}
```
</Aside>
